# Чек-лист — форма авторизации (SauceDemo)

**Объект:** страница входа https://www.saucedemo.com
**Автор:** Боголюбов Александр

Чек-лист — это список проверок без подробных шагов, для быстрого прохода по функциональности. Отмечай `[x]` пройденные пункты.

## Вёрстка и интерфейс
- [ ] Логотип и название отображаются корректно
- [ ] Присутствуют поля «Username» и «Password» и кнопка «Login»
- [ ] В полях есть подсказки (placeholder)
- [ ] Элементы не съезжают и не перекрываются при изменении размера окна
- [ ] Страница корректно отображается на мобильном разрешении

## Позитивные сценарии
- [ ] Вход с валидными логином и паролем (`standard_user`)
- [ ] После входа происходит переход в каталог
- [ ] Вход срабатывает по нажатию Enter, а не только по кнопке

## Негативные сценарии
- [ ] Пустой логин — показывается «Username is required»
- [ ] Пустой пароль — показывается «Password is required»
- [ ] Неверный логин/пароль — показывается «do not match any user»
- [ ] Заблокированный пользователь — показывается «locked out»
- [ ] Ввод только пробелов в поля не считается валидным
- [ ] Логин с другим регистром (`Standard_User`) не проходит

## Поля и валидация
- [ ] Пароль скрыт точками при вводе
- [ ] Текст в поля можно вставить через копирование (paste)
- [ ] Пробелы в начале/конце логина обрабатываются корректно
- [ ] Очень длинное значение в поле не ломает вёрстку
- [ ] Спецсимволы и SQL-подобный ввод (`' OR 1=1 --`) не вызывают ошибку приложения

## Безопасность
- [ ] Пароль не передаётся и не виден в адресной строке (URL)
- [ ] Сообщение об ошибке не подсказывает, какое именно поле неверное (защита от перебора)
- [ ] После выхода (Logout) возврат по кнопке «назад» не возвращает в каталог

## Юзабилити
- [ ] Переход между полями работает по клавише Tab
- [ ] Сообщения об ошибке понятны и видны пользователю
- [ ] Курсор по умолчанию стоит в поле логина (удобно)

---

**Итого:** 25 проверок по форме авторизации — вёрстка, позитив, негатив, валидация, безопасность, юзабилити.
