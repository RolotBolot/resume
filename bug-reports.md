# Баг-репорты — SauceDemo (Swag Labs)

**Объект тестирования:** https://www.saucedemo.com
**Автор:** Боголюбов Александр

Дефекты найдены при тестировании под учётной записью `problem_user` (пароль `secret_sauce`) — это режим, в котором приложение специально содержит ошибки для отработки навыка.

> **Перед публикацией:** воспроизведи каждый баг сам и приложи **свои** скриншоты вместо плейсхолдеров `[скриншот]`. Это правило тестировщика — баг публикуется только после личного воспроизведения.

## BUG-01 — Все товары в каталоге отображаются с одинаковой картинкой

| Поле | Значение |
|------|----------|
| **ID** | BUG-01 |
| **Severity** | Major |
| **Priority** | Medium |
| **Окружение** | Chrome 126, Windows 10, https://www.saucedemo.com, пользователь `problem_user` |

**Предусловия:** выполнен вход под `problem_user`.

**Шаги воспроизведения:**
1. Войти под `problem_user` / `secret_sauce`.
2. Открыть страницу каталога `/inventory.html`.
3. Сравнить изображения всех 6 товаров.

**Ожидаемый результат:** у каждого товара своя картинка, соответствующая названию.

**Фактический результат:** все 6 товаров отображаются с одним и тем же изображением.

**Вложения:** `[скриншот каталога]`

## BUG-02 — Сортировка товаров не работает

| Поле | Значение |
|------|----------|
| **ID** | BUG-02 |
| **Severity** | Major |
| **Priority** | High |
| **Окружение** | Chrome 126, Windows 10, https://www.saucedemo.com, пользователь `problem_user` |

**Предусловия:** выполнен вход под `problem_user`, открыт каталог.

**Шаги воспроизведения:**
1. На странице каталога открыть выпадающий список сортировки.
2. Выбрать **Price (low to high)**.
3. Посмотреть на порядок товаров.

**Ожидаемый результат:** товары перестраиваются по цене от меньшей к большей.

**Фактический результат:** порядок товаров не меняется, сортировка не применяется.

**Вложения:** `[скриншот до и после выбора сортировки]`

## BUG-03 — Кнопка «Remove» не убирает товар из каталога

| Поле | Значение |
|------|----------|
| **ID** | BUG-03 |
| **Severity** | Critical |
| **Priority** | High |
| **Окружение** | Chrome 126, Windows 10, https://www.saucedemo.com, пользователь `problem_user` |

**Предусловия:** выполнен вход под `problem_user`, открыт каталог.

**Шаги воспроизведения:**
1. Нажать **Add to cart** у товара — кнопка меняется на **Remove**, счётчик корзины растёт.
2. Нажать **Remove** у этого же товара.

**Ожидаемый результат:** товар убирается из корзины, счётчик уменьшается, кнопка снова становится **Add to cart**.

**Фактический результат:** нажатие **Remove** не срабатывает — товар остаётся в корзине, состояние кнопки не меняется.

**Вложения:** `[скриншот корзины со счётчиком]`

**Итого:** 3 дефекта разной критичности (Major / Critical) с полным описанием шагов и окружения.
